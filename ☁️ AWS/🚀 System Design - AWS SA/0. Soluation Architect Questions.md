# Lambda Concurrency Controls 🎯

### ⚡ Key Concept: Concurrency

👉 **Concurrency = number of Lambda executions running at the same time**

- 1 request → concurrency = 1
- 500 requests → concurrency = 500

## A. Account-level concurrency

- Default limit (e.g., ~1000 per region)
- Shared across all Lambdas

## B. Reserved Concurrency

- Set limit for a specific function
- Guarantees capacity

Example:

Function A → reserved 300  
Function B → uses rest

## C. Provisioned Concurrency ⚡

- Pre-warmed instances
- No cold starts - 
    - New instances → **cold start**

👉 Useful for:
- Low latency APIs
- Critical services
