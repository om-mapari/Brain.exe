# 1пёЏвѓЈ What is a Generator

- A function that **yields values lazily**
- Uses `yield`
- Remembers state
- Does not keep all data in memory


```python
def gen():
    yield 1
    yield 2
    yield 3

g = gen()
print(g)        # <generator object gen at 0x...>
print(list(g))  # [1, 2, 3]
print(list(g))  # []  (exhausted)
```

# 2пёЏвѓЈ Lazy Evaluation Concept

Generator produces values **only when asked**

```python
def squares(n):
    for i in range(n):
        print(f"computing {i}^2")
        yield i * i

sq = squares(3)
print(next(sq))  # computing 0^2  -> 0
print(next(sq))  # computing 1^2  -> 1
print(next(sq))  # computing 2^2  -> 4
# next(sq)       # StopIteration
```

# 7пёЏвѓЈ Generator Pipeline (chaining)

```python
def nums():
    for i in range(6):
        yield i

def evens(stream):
    for x in stream:
        if x % 2 == 0:
            yield x

def mul2(stream):
    for x in stream:
        yield x * 2

pipe = mul2(evens(nums()))
print(list(pipe))  # [0, 4, 8]

# OR
print(next(pipe))  # 0
print(next(pipe))  # 4

```

Breakdown:

- nums в†’ `[0,1,2,3,4,5]`
- evens в†’ `[0,2,4]`
- mul2 в†’ `[0,4,8]`


```python
nums() в†’ 0 в†’ evens keeps в†’ mul2 в†’ 0*2 в†’ 0
nums() в†’ 1 в†’ evens drops
nums() в†’ 2 в†’ evens keeps в†’ mul2 в†’ 2*2 в†’ 4

```


#### в–¶пёЏ First call: `next(pipe)`

1. `mul2()` asks its source for a value (`next(evens(...))`)
2. `evens()` asks its source for a value (`next(nums())`)
3. `nums()` runs until first `yield`

```python

nums() yields 0 # 0

```

4. `evens()` checks `0 % 2 == 0` в†’ True, so it yields `0`
5. `mul2()` takes `0` and yields `0 * 2`

```python

next(pipe) prints 0 # 0

```

---

#### в–¶пёЏ Second call: `next(pipe)`

1. `mul2()` again asks `evens()` for next
2. `evens()` asks `nums()` for next
3. `nums()` resumes from where it paused and yields `1`

```python

nums() yields 1 # 1

```

4. `evens()` checks `1 % 2 == 0` в†’ False, so it **skips**
5. `evens()` asks `nums()` again
6. `nums()` resumes and yields `2`

```python

nums() yields 2 # 2

```

7. `evens()` checks `2 % 2 == 0` в†’ True, so yields `2`
8. `mul2()` takes `2` and yields `2 * 2`

```python

next(pipe) prints 4 # 4

```

#### So the real execution looks like this:

```python

nums() в†’ 0 в†’ evens keeps в†’ mul2 в†’ 0*2 в†’ 0
nums() в†’ 1 в†’ evens drops
nums() в†’ 2 в†’ evens keeps в†’ mul2 в†’ 2*2 в†’ 4

```

# 9пёЏвѓЈ Two-way Generator using `send()`

```python
def talk():
    msg = yield "Start"
    yield f"You said {msg}"
    msg2 = yield "Ask again"
    yield f"Second msg {msg2}"

t = talk()
print(next(t))      # Start
print(next(t))      # Ask again  вќЊ Wrong, because we skipped send

# Correct usage в†“
t = talk()
print(next(t))      # Start
print(t.send("Hi")) # You said Hi
print(t.send(123))  # Second msg 123
```

#### What really happens one by one

1. `next(t)`
    - Generator starts
    - Hits `yield "Start"`
    - Returns `"Start"`
    - You print it в†’ `Start`

2. `t.send("Hi")`
    - `"Hi"` goes into `msg`
    - Generator resumes
    - Hits `yield f"You said {msg}"` в†’ `"You said Hi"`	
    - Returns `"You said Hi"`	
    - You print it в†’ `You said Hi`

3. `t.send(123)`	
    - `123` goes into `msg2`	
    - Generator resumes	
    - Next `yield` it reaches is `yield "Ask again"`	
    - Returns `"Ask again"`	
    - You print it в†’ `Ask again`

4. After that, the generator is **paused at**

```python
yield f"Second msg {msg2}"
```

if you never called `next()` again, so it won't print `"Second msg 123"`

5. If you call one more:

```python

print(next(t)) # Second msg 123

```

> [!Note]
>  `send(value)` only works after the generator has started using `next()`
